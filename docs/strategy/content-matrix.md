# Jukkai content matrix (Phase 3 skeleton)

**Phase 3 deliverable, produced July 2, 2026, alongside `docs/strategy/sitemap.md`.** The
left half (strategy/IA) is filled here; the right half (final copy, meta strings,
schema, live status) fills in Phase 4. `docs/strategy/foundation.md` v1.5 is truth;
`docs/strategy/messaging.md` binds registers, proof themes, vocabulary, and every
[VALIDATION-NEEDED] item. Nothing in this file is final page copy.

## How to read this file

- **Table A** is the IA/strategy half: one row per surface, launch pages first,
  then reserved post-launch rows. Title patterns and H1 territories are patterns;
  exact strings are Phase 4 work inside these rails.
- **Table B** is the proof/migration/validation half, keyed by the same Page
  names.
- **The provisional project inventory** catalogues every old-site project page as
  a candidate proof row. Blanket rule: for every row in that table, **publication
  permission and representativeness are validation-needed with Crystelle (S1)**;
  nothing there may be treated as publishable or representative until she says so.
- Decision keys: `S1` = questions-for-crystelle blocking-for-sitemap #1 (launch
  portfolio). `C1..C9` = blocking-for-copy items. `L1..L6` = blocking-for-launch
  items. `V1..V8` = messaging §9 validation list.
- Validation status vocabulary:
  - `draft-ready`: Phase 4 may draft and freeze copy (claims canon applies).
  - `draft-gated`: Phase 4 may draft with placeholders; copy freeze blocked on
    the referenced decision(s).
  - `proof-pending`: structure and copy rails ready; proof rows await S1.
  - `reserved`: post-launch; do not build at launch, keep the row.

## Notes for Phase 4

1. **Row-filling order.** Guide coût and service page first (P0, contested);
   espaces professionnels next (P1, launches on process even before references
   clear); shop page copy can draft early but freezes late (largest validation
   surface: C8, C9, L1, L4, L5, V1, V2, V6, V8).
2. **The matrix's validation-status column is the freeze gate.** `draft-gated`
   rows may ship placeholder copy internally; silent promotion to final is a
   defect (messaging §10.10).
3. **Vocabulary bans are absolute** (messaging §8). A banned word in a draft is a
   defect, not a style choice. The thesis line appears once, shop page only.
4. **Claims canon only** (foundation §2). Never blur 30 (career) and 2012
   (studio). Bridge claims stay vision-tense until C7 produces a documented
   placement.
5. **Quote hygiene**: any French verbatim reused from `docs/research/audience.md` must be
   spot-checked at source; testimonials need documented permission (C4).
6. **Not in this matrix but launch-weight**: GBP + citations checklist, page-by-
   page 301 map (built from Table B's Old URL column + the inventory), GA4/Search
   Console with form-submission conversion, external profile updates. Phase 4
   owns their specs (analytics, schema, redirect map) as separate artifacts.
7. **Schema (Phase 4 decisions flagged here)**: LocalBusiness sitewide;
   ArtGallery-equivalent on `/art-shop/` (the structural answer to the
   banned-vocabulary tension); FAQPage on the three acquisition surfaces.
8. **Geolocated SERP spot-check** on the two launch clusters before the SEO spec
   freezes (keywords §1 limitation); check whether IDKREA came back online
   (competitors: threats) at IA freeze and pre-launch.
